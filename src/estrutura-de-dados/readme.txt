

  A descrição dos códigos aqui é, eles são estrutura de dados assim como
no outro diretório, porém eles são bem contidos e não compartilham, ou
depededem de outros módulos. Todos componentes estão num arquivo, ou 
na pasta do arquivo, este que contém outros arquivos, que serão incluídos
em tempo de compilação no arquivo principal, ou seja, o mesmo arquivo.

  Por que algo assim? Bem, eu vi que copiar outro coleção para colocar em
qualquer código feito, é preciso de uma alteração entre os módulos, assim
como instruções de compilações monstruosas, por isso colocar tudo no
mesmo balaio, assim só muda coisas do arquivo quando precisar desta 
estrutura com um tipo diferente.

  As estrutura aqui também são bem diferentes e simples em funcionalidade
da coleção de estruturas de dados original. Além de estarem com a forma
mais idiomatica em C, não aceitam dados genéricos o que é algo do 
original, esteja implementado ou não ainda.
